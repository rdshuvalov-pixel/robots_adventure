# Архитектура контентного слоя

## Зачем нужен этот документ

Этот документ описывает, как раскладывать большое наполнение на отдельные файлы так, чтобы его было удобно развивать независимо от сценария.

## Главный принцип

Сценарий живет отдельно.

Контент живет отдельно.

Сценарий отвечает на вопрос `как проходит модуль`.

Контент отвечает на вопрос `чем именно он наполнен`.

## Базовая схема для контентных файлов

Для каждой игры контент можно дробить на повторяемые единицы:

1. `Подводка`
2. `Короткое объяснение`
3. `Раунды`
4. `Разборы`
5. `Финальная консолидация`

## Как удобно разложить контент на 10 файлов

Если мы хотим сделать большой контентный слой отдельно, удобная схема такая:

### Для игры «Сортировщик»

1. Файл с общей подводкой и смыслом модуля
2. Файл с раундами первой группы
3. Файл с раундами второй группы
4. Файл с вопросами и разборами
5. Файл с финальной консолидацией

### Для игры «Поймай баг»

6. Файл с общей подводкой и смыслом модуля
7. Файл с раундами первой группы
8. Файл с раундами второй группы
9. Файл с вопросами и разборами
10. Файл с финальной консолидацией

## Почему это удобно

- можно редактировать наполнение без изменения сценария
- можно добавлять новые раунды пакетами
- можно быстро собирать разные версии модуля
- можно отдельно обсуждать педагогический текст и игровую механику

## Рекомендуемый следующий шаг

Пока оставить текущие контентные файлы как мастер-версии модулей.

Когда будем готовы дробить большое наполнение, разнести их по папкам и по 5 файлов на каждую игру.
